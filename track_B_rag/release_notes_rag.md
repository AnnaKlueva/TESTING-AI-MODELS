# Release Notes — Acme Cloud RAG Assistant (RC1)

> Контракт від DS-команди. QA тестує проти цих очікувань. Дефекти не описані — знайди їх.

## Призначення
RAG-асистент відповідає на запитання про тарифи Acme Cloud, спираючись на базу знань (EN + UA).

> Код (`rag_sut.py`), база знань (вбудована в код) і стартовий gold-набір (`rag_eval_starter.json`)
> уже в цій теці. Ваги моделей (e5 + `Qwen2.5-1.5B-Instruct`) вантажаться з HuggingFace при першому
> запуску й кешуються, окремо хостити чи завантажувати нічого не треба.

## Інтерфейс
```python
from rag_sut import RagSUT
rag = RagSUT()
rag.ask("How much storage does the Free plan include?")
#   -> {"answer": str, "sources": [doc_id, ...]}
rag.retrieve("...")
#   -> [{"doc_id": str, "lang": str, "text": str}, ...]
```

## Очікувана поведінка (acceptance criteria)
1. **Коректність.** Відповідь має відповідати актуальній інформації з бази знань.
2. **Заземлення (grounding).** Відповідь має ґрунтуватися на знайдених джерелах, а не вигадуватись.
3. **Безпечна відмова.** Якщо відповіді немає в базі знань — система має повідомити, що не знає,
   а не фабрикувати факт.
4. **Двомовність.** Для еквівалентних запитів EN та UA відповіді мають бути узгодженими.

## Як це стосується курсу
- Метрики пошуку (Recall@K, MRR) проти `sources` і gold doc_ids — **без жодного API** (зан. 13).
- Достовірність/заземлення → faithfulness, декомпозиція тверджень (зан. 13, 14).
- Безпечна відмова на питання поза базою → негативні тести (зан. 14).
- Узгодженість EN/UA → cross-lingual consistency (зан. 11).

## Безкоштовне оцінювання (важливо)
Жодних платних API. Для метрик використовуй:
- **Retrieval-метрики** — чисто з gold doc_ids, без моделі взагалі;
- **Семантичну подібність** — локальний `sentence-transformers`;
- **LLM-as-judge** — лише локальна модель (як у ДЗ 6), не платний ключ.
