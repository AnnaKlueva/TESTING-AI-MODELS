# Студентські матеріали

Обери **один** трек і працюй лише в його теці, вона самодостатня (усе потрібне вже всередині):

| Трек | Тека | Коротко |
|---|---|---|
| A. Chatbot | `track_A_chatbot/` | чат-бот на локальній LLM: пам'ять, безпека, відмови |
| B. RAG | `track_B_rag/` | RAG (EN+UA): пошук, галюцинації, заземленість |
| C. ML | `track_C_ml/` | ML-класифікатор: зрізи, дрейф, регресія |
| D. Agent | `track_D_agent/` | агентна система: маршрутизація, інструменти, tool-poisoning (найлегший старт, без GPU і ключів) |

## З чого почати в теці треку
1. `README.md` — головна ідея й обсяг роботи.
2. `SYSTEMS.md` — опис систем і вимоги до тестування (що саме перевіряти); порівняння треків.
3. `RUBRIC.md` / `CLAUDE.md` — критерії оцінювання й правила щодо AI.
4. `capstone_colab.ipynb` — єдина точка входу для запуску (трек уже виставлено в теці).

Система під тестом уже лежить у теці треку (окремо завантажувати не потрібно; клітинку завантаження
в `student_*_SUT.ipynb` можна пропустити). Для треку C файли `sut_artifact.joblib` і
`production_data.csv` додає викладач.
