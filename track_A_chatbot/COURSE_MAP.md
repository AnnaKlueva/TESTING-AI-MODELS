# Мапа покриття курсу проєктом

Курсовий проєкт — це **міні-версія всього курсу**: кожна дільниця роботи спирається на конкретний
блок/заняття. Використай цю таблицю у `test_strategy.md` і на захисті, щоб показати, що твоя робота
демонструє курс цілком, а не лише один трек.

| Блок курсу | Заняття | Дільниця проєкту | Артефакт |
|---|---|---|---|
| **1. Основи AI Quality** | 1–3 | Матриця ризиків + таксономія фейлів (root cause) | `test_strategy.md` §3, `reports/results.md` |
| **2. Класичні ML-моделі** | 4–6 | Метрики по зрізах, регресія, дрейф (трек C) | `tests/test_eval.py` |
| **3. Оцінювання LLM** | 7–11 | Недетермінізм (pass-rate), LLM-as-judge, UA/EN, multi-turn (трек A) | `src/metrics/`, датасет |
| **4. RAG і пошук** | 12–14 | retrieval-метрики, faithfulness, галюцинації (трек B) | `tests/test_eval.py` |
| **5. Безпека / Red-teaming** | 15 | adversarial-кейси, prompt-injection, tool poisoning | `tests/test_redteam.py` |
| **+ Агентні системи** | 16 (практикум «Agentic QA») | routing, tool call accuracy, trajectory, task completion (трек D) | `tests/`, agent-траси |
| **6. Автоматизація і фреймворки** | 17–18 | unit/mock/integration, DeepEval, `generations.json`-гейт | `tests/`, `run_eval` |
| **7. Production і MLOps** | 19–20 | розглядається в лекціях; у проєкті — відтворюваний офлайн `run_eval` як основа, яку запускав би CI | `run_eval`, `generations.json` |
| **8. Процеси і команда** | 21–23 | тест-стратегія, traceability, презентація й захист | `test_strategy.md`, слайди |

## Мінімальне покриття (обов'язкове для будь-якого треку)
Незалежно від обраного треку, кожен проєкт **має** зачепити ці наскрізні дільниці (це і робить його
«міні-курсом»):

- **Блок 1** — матриця ризиків + таксономія (без цього не буде критерію 1);
- **Блок 5** — щонайменше кілька **adversarial** кейсів (safe refusal / injection), навіть для ML;
- **Блок 6** — відтворюваний офлайн-`run_eval` над `generations.json`;
- **Блок 8** — тест-стратегія + traceability (ризик → кейс → результат) + захист.

Специфіка треку (Блок 2/3/4 або 16) — це «глибина», а наскрізні дільниці — «ширина».
Разом вони показують курс цілком.

## Як показати це на захисті (7–10 хв)
1. Одна точка ризику з матриці (Блок 1) →
2. кейси в датасеті, що її покривають (Блок 5/6) →
3. метрика треку, що її ловить (Блок 2/3/4/16) →
4. знайдений дефект + root cause (Блок 1) →
5. як це стало б quality-gate у CI (Блок 6/7).
