# Release Notes — Acme ML Classifier (RC1)

> Контракт від DS-команди. QA тестує проти цих очікувань. Дефекти не описані — знайди їх.

## Призначення
Табличний класифікатор кредитного ризику (sklearn-пайплайн). Передається як заморожений артефакт
`sut_artifact.joblib` разом із міченим production-набором `production_data.csv`.

## Інтерфейс
```python
import joblib, pandas as pd
bundle = joblib.load("sut_artifact.joblib")
pipe, features = bundle["pipeline"], bundle["features"]
prod = pd.read_csv("production_data.csv")       # містить справжню мітку 'default'
pred = pipe.predict(prod[features])
```
> Файли `sut_artifact.joblib` і `production_data.csv` **уже лежать у цій теці** (в репозиторії),
> завантажувати нічого не треба.

## Вимоги до тестування (що перевірити)
1. **Якість по зрізах** — чи не провалюється модель на окремих сегментах (напр. `region`, вікові групи), а не лише загалом.
2. **Пастка точності** — чи адекватні precision/recall при дисбалансі класів.
3. **Дрейф даних** — чи відрізняється розподіл production від тренувального.
4. **Надійність ознак** — чи немає ознаки, що «лікає» і не генералізується.
5. **Регресія** — порівняння з baseline, де доречно.

## Як це стосується курсу
Метрики по зрізах і пастка точності (зан. 4), дрейф (зан. 6), таксономія фейлів даних (зан. 3).
